# Phase 2 — Sandbox — @upstash/box + local bash() real-machine adapter
> Ship the two built-in `SandboxFactory`s — the isolated `@upstash/box` sandbox and the local in-process `bash()` real-machine adapter — both satisfying one `SessionEnv` contract with workspace confinement, lazy provisioning, and by-name box resolution. Design-of-record: [06 — Roadmap](../../design/06-phase-roadmap.md) (Phase 2) + [08 §3 boundary](../../design/08-conventions-and-execution-boundary.md#3-the-execution-boundary-core-philosophy) + [04 — Durable Engine](../../design/04-durable-engine.md). Decisions: [D1–D19](../../design/07-risks-and-decisions.md) (esp. [D7](../../design/07-risks-and-decisions.md), R5/R6).

## Goal & scope

Build the execution layer that the durable engine (P4) dispatches tool calls into. Two built-in `SandboxFactory` implementations, both reached **only from `"use node"` engine actions**, both confined to one per-session workspace folder, both round-tripping the 9-method `SessionEnv` contract:

1. **`@upstash/box` sandbox** (`convex/sandbox/upstashBox.ts`) — a `SandboxApi` over the `@upstash/box` SDK, wrapped by the **ported `createSandboxSessionEnv`** for path resolution, parent-dir creation, and abort checks. Lazy box provisioning, by-name resolution (`Box.list`/`Box.get`), warm/cold caching that survives within an action but never across actions, box-gone re-provisioning, and `SandboxOperationUnsupportedError` for unservable methods.
2. **Local in-process `bash()` real-machine adapter** (`convex/sandbox/localBash.ts` + ported `bash()`/`createBashSessionEnv` in `convex/sandbox/sessionEnv.ts`) — the **real-machine** target ([D7](../../design/07-risks-and-decisions.md)) that makes `BashFactory`/`BashLike` a *consumed* contract over Node `fs`/`child_process`, satisfying the same `SessionEnv` + confinement rules.

**In scope:** the `SessionEnv` adapters, the two factory constructors (`upstashBox()`, `localBash()`/`bash()`), the ported sandbox scaffolding (`createSandboxSessionEnv`, `createCwdSessionEnv`, `writeFileCreatingParents`, path-normalize helpers, `bash()`/`bashFactoryToSessionEnv`/`createBashSessionEnv`), the `composeTimeoutSignal`/`abortErrorFor` abort primitives this phase needs, the `sandboxName` resolution key, and the `vitest` round-trip tests for both built-ins.

**Out of scope (later phases):** the engine actions that *call* these factories (`engine/setup` resolve-only, `engine/dispatchTools` execute) land in **P4**; `session.shell()`/`harness.shell()` event envelope wiring lands in **P6** (the ported `src/runtime/shell.ts`); the `skills` catalog and on-demand workspace reads land in **P10**. This phase delivers the seam and proves both built-ins satisfy it.

## Dependencies

- **P1 (done)** — provides the `SessionEnv`, `SandboxFactory`, `BashFactory`, `BashLike`, `FileStat`, `ShellResult` types in [`src/runtime/types.ts`](../../../src/runtime/types.ts) and `SandboxOperationUnsupportedError` in [`src/runtime/errors.ts`](../../../src/runtime/errors.ts). This phase *consumes* those types; it does not redefine them.
- No dependency on P3/P4/P5 — the factories are pure adapters with no engine/session-store coupling. P4 depends on **this** phase (resolves the `SessionEnv` at `setup`, dispatches box-bound tools at `dispatchTools`).

## Hardened-contract obligations

This phase implements the **execution-boundary** half of [08 §3](../../design/08-conventions-and-execution-boundary.md#3-the-execution-boundary-core-philosophy) and the sandbox-lifecycle rules. Specific rules to honor:

- **[08 §3 — "The sandbox is one designated folder"]** — every `SessionEnv` is scoped to one workspace `cwd`. `createSandboxSessionEnv`/`createCwdSessionEnv` resolve relative paths against `cwd` and **reject `../` escapes**; `mkdir`/`rm`/`readdir`/parent-creation all stay inside the workspace. `ShellOptions.cwd` resolves *within* the workspace, never above it.
- **[08 §3 — "Which Convex functions may touch the sandbox"]** — both factories are reachable **only from `"use node"` actions**. `upstashBox.ts` and `localBash.ts` MUST carry `"use node"`; `sessionEnv.ts`/`abort.ts` MUST stay pure (no box/fs import) so they remain V8-safe and importable from anywhere.
- **[08 §3 — `sandboxName` (resolution key)]** — `` sandboxName = `${ctx.id}:${instanceId}:${harnessName}` ``. The box is resolved by name (`Box.list` → filter `BoxData.name === sandboxName` → `Box.get(id)`) on each **cold** action; the in-process handle cache is per-action only and never relied on across actions (boxes outlive stateless actions; handles do not). **R5.**
- **[08 §3 — lazy provisioning]** — **no box is created until the first `exec`/fs operation**. `upstashBox()` returns a factory whose `createSessionEnv` only *prepares* a lazy `SandboxApi`; the actual `Box.create`/`Box.get` fires on the first method call. `setup` is "resolve only" ([D13](../../design/07-risks-and-decisions.md)) — it must not force provisioning.
- **[08 §3 — box-gone failure handling]** — a previously-resolved box that no longer exists triggers re-provisioning (or a typed failure), never a wedged run on a stale handle.
- **[06 P2 / 08 §3 — `SandboxOperationUnsupportedError`]** — a partial box impl that cannot service a `SessionEnv` method raises this typed error; it **never silently no-ops**. (E.g. `rm` with an option the backend can't honor; a confinement-incapable real-machine adapter.)
- **[08 §4.2 — action budgets]** — default box `exec` `timeoutMs` = **30 s** is the adapter default; **`timeoutMs` rounds *up* to box-second granularity, never down** (R5). `exec` honors both `timeoutMs` (primary) and `signal` (pre/post abort checks centralized in `createSandboxSessionEnv`, per the ported comment).
- **[08 §4.11 / D19 — env redaction]** — *consumed* in P6, but the adapter contract this phase locks must keep redaction possible: `env.exec` receives the **real** `env` values (the redaction happens in the shell envelope, not the adapter). Document that the adapter must never log/persist `env` values itself.
- **[06 P2 — concurrency gate]** — concurrent same-session operations serialize via the `agentRequests`/`sessions.state` gate (owned by P4/P6). This phase's adapters are not internally re-entrant-safe against the *same box handle* and rely on that upstream gate; note it, don't re-implement it.

## Acceptance

Start from 06 P2's acceptance bar; each bullet is a concrete test in the two `*.test.ts` files.

- [ ] **Box round-trip (06 P2):** a vitest runs `exec("echo hi")` and round-trips `writeFile`/`readFile`/`stat`/`readdir`/`rm` through a (fake) box — all assertions pass. **PASS/FAIL:** the round-trip values match.
- [ ] **Lazy provisioning (06 P2 / 08 §3):** `createSessionEnv({id})` provisions **no** box; the **first** `exec`/fs op triggers exactly one `Box.list`+`get` (or `create`). **FAIL** if `create`/`get` fires during factory construction.
- [ ] **By-name resolution + warm/cold (08 §3, R5):** a warm box (named `${ctx.id}:${instanceId}:${harnessName}`) is reused via `get`; within one action the handle is cached (no re-`list`); a cold factory re-resolves by name. **FAIL** if a cold action relies on a surviving in-process handle.
- [ ] **Box-gone (08 §3):** a vanished cached handle re-provisions rather than wedging. **FAIL** if the second op throws the stale-handle error unrecovered.
- [ ] **`../`-escape rejected (06 P2 / 08 §3):** every `SessionEnv` method rejects `../`, `../../`, `a/../../b`, and an absolute path outside `cwd`. **FAIL** if any resolves to a path at/above the workspace parent. *(Catches the silent-`..`-collapse gap in flue's `normalizePath`.)*
- [ ] **`SandboxOperationUnsupportedError` (06 P2 / 08 §3):** a partial impl raises the typed error (never a silent no-op or generic `Error`). **FAIL** if the method returns `undefined` or throws a non-`CoveError`.

## Risks & gotchas

- **`"use node"` placement (Convex trap).** The `"use node"` directive must be the **very first statement** of `upstashBox.ts`/`localBash.ts`, before the reference-header *can* follow but the directive must precede any import. A module that imports `@upstash/box` or `child_process` from the V8 runtime (no `"use node"`) fails at deploy, not `tsc`. Keep `sessionEnv.ts`/`abort.ts` import-clean so they can be imported by both runtimes.
- **Silent `..` collapse (the real bug).** flue's `normalizePath` *pops* on `..` and never errors — `/workspace/../etc` becomes `/etc` silently. The roadmap demands **rejection**. Do not assume the ported helper already guards this; task 3 adds the explicit boundary check and the test proves it. This is the single most important correctness delta of the phase.
- **Stateless actions vs box handles (R5).** Convex actions are stateless and may run on different machines; the in-process box handle does **not** survive across actions. Resolve by name on every cold action. The per-action cache is an optimization within one action only — never persist or assume it.
- **By-name resolution races.** `Box.create({name})` is not guaranteed atomic against a concurrent same-name create. Rely on the upstream `agentRequests`/`sessions.state` serialization gate (P4/P6) to ensure one provisioning at a time; document the assumption rather than adding locking here.
- **`timeoutMs` granularity.** Box exec timeouts are second-granular; **always `Math.ceil`** — rounding down would kill long tool calls early and violate the LLM-bash-tool deadline contract.
- **base64 `bash -l` round-trip.** Wrap the command base64 to survive quoting/newlines, and decode on the box (`base64 -d`). `bash -l` (login shell) is required so the box's PATH/env profile is loaded (parity with flue's exec). Watch for environments where `base64` isn't on PATH before the profile loads — fall back to `bash -lc` with the raw command only if base64 is unavailable, and test both.
- **stdout/stderr split fidelity.** `@upstash/box`'s `Run`/`ExecResult` may merge stdout+stderr. Map what the SDK gives (`Run.result`/`ExecResult.output` → stdout, `ExecResult.error` → stderr when present); never fabricate a separate stderr if the backend doesn't provide one — leave `stderr: ''`. (Mirrors the `FileStat`-no-fabrication rule in [`types.ts`](../../../src/runtime/types.ts).)
- **Abort across the action boundary (R2).** The centralized pre/post `signal?.aborted` checks in `createSandboxSessionEnv` are the *only* abort fidelity a signal-blind box gets mid-exec; a tool already in-flight on the box won't see the cancel until its `exec` returns. This is expected (documented weakened mid-tool abort); `timeoutMs` bounds the worst case. Do not try to add mid-flight cancel the SDK doesn't support.
- **`createFlueFs` deferral.** Resist porting `createFlueFs`/the `CoveFs` adapter now — it belongs to the P6 `harness.fs`/`session.fs` facade. Porting it here pulls in surface this phase doesn't test and risks drift.
- **Replay determinism is *not* this phase's job.** These adapters are side-effecting executors; idempotency/replay de-dup lives in `dispatchTools`/`appendToolResult` (P4, [08 §4.5](../../design/08-conventions-and-execution-boundary.md#45-tool-rebuild-from-frozen-descriptors)). Do not add replay logic to the `SessionEnv` — keep it a thin executor so P4 owns the journal.
