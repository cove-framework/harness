# Phase 8.5 — CLI + codegen
> The Convex-native `cove` binary (`dev`/`build`/`deploy`) replacing `@flue/cli`, plus `defineAgentRegistry` + `defineWorkflow` codegen into the deployed app wiring with build-time validation. Design-of-record: [06 — Roadmap](../../design/06-phase-roadmap.md) + [05 — Public API & SDK](../../design/05-public-api-and-sdk.md), [08 — Conventions](../../design/08-conventions-and-execution-boundary.md). Decisions: [D1–D19](../../design/07-risks-and-decisions.md) (esp. [D6, D12, D18](../../design/07-risks-and-decisions.md)).

## Goal & scope

Replace flue's filesystem-module addressing + Vite/Hono build pipeline (`@flue/cli`) with a **Convex-native** authoring + wiring pipeline:

1. **`cove` binary** exposing `cove dev` / `cove build` / `cove deploy` over the Convex toolchain (wrapping `convex dev` / `convex deploy`), with arg/flag parsing, env loading, and `[cove]`-prefixed diagnostics.
2. **`defineAgentRegistry` codegen** → emits the Convex app wiring (`convex/agentRegistry.ts` resolver + the `convex/http.ts` router entry from P8) from the **declared** registry map. The registry is the single source of agent addressing (replacing flue's `agents/<name>.ts` filename addressing, [D6](../../design/07-risks-and-decisions.md)).
3. **`defineWorkflow` registry/codegen** → `defineWorkflow((ctx) => result)` returns a `WorkflowHandler`, registered in a Convex-app-bound `convex/workflowRegistry.ts`; codegen wires the declared handlers to their `POST /workflows/:name` routes (the routes themselves land in P8, [D18](../../design/07-risks-and-decisions.md)).
4. **Build-time validation** of the agent registry + workflow registry + provider + durability config: name regex, uniqueness, brand markers, declared-subagent existence, declared provider — failing **before deploy** with a `[cove]` diagnostic.
5. **Deferred build-time skill packaging (m3)** — reintroduce the `SkillReference` / `PackagedSkillDirectory` *authoring/packaging* seam ([08 §5](../../design/08-conventions-and-execution-boundary.md#5-dropped--obsolete-explicit)). Scope here is the **producer contract + a packaging entrypoint that emits catalog-import payloads**, feeding the P10 skills catalog — NOT runtime skill resolution (that stays catalog-table per [D13](../../design/07-risks-and-decisions.md)).

**Explicit non-goals (deferred, per [06 P8.5](../../design/06-phase-roadmap.md#-phase-85--cli--codegen) cross-refs):**
- **`cove add` / blueprint scaffolding (m7)** — the CLI ships `dev`/`build`/`deploy` **only**. The blueprint generator ([`cli/lib/blueprint-index.ts`](../../../../flue/packages/cli/src/lib/blueprint-index.ts)) is *not* ported this phase. Leave a stubbed command that prints a `[cove]` "deferred" notice; do not wire `BLUEPRINTS`/`MiniSearch`.
- **Cloudflare target** — dropped ([D12](../../design/07-risks-and-decisions.md)); `--target` does not exist. Convex is the only deploy backend.
- **Vite/Hono bundling, `flue run` / `flue connect` / `flue logs` / `flue docs` / `flue init`** — Convex's own toolchain owns dev/deploy; the local IPC harness and DS-log tailing have no Convex analogue.

## Hardened-contract obligations

This is a **build-time/wiring** phase — it touches none of the durable-loop runtime contracts (§4.1–§4.11), but it must **preserve the boundaries those contracts assume** so the generated wiring cannot weaken them:

- **[08 §1 Naming](../../design/08-conventions-and-execution-boundary.md#1-naming--namespace)** — every emitted + authored file uses `Cove*` brand types, `[cove]` error prefix, `cove_error` code, `__coveCreatedAgent` brand. No `Flue`/`flue`/`pi` identifier survives in generated output.
- **[08 §1 / §5 app-bound surface](../../design/08-conventions-and-execution-boundary.md#5-dropped--obsolete-explicit)** — `defineAgentRegistry` and `defineWorkflow` are **Convex-app-bound**: they live under `convex/` (`convex/agentRegistry.ts`, `convex/workflowRegistry.ts`, `convex/workflows/`) and are exported from the app/CLI surface, **never** re-exported from the `@cove/runtime` barrel ([`src/runtime/index.ts`](../../../src/runtime/index.ts)), which must stay V8-safe and Convex-free. Codegen must NOT add a runtime-barrel export.
- **[08 §2 Reference-header](../../design/08-conventions-and-execution-boundary.md#2-reference-header-convention)** — every new source file opens with the origin header. Codegen-**emitted** files additionally carry an `// Auto-generated by cove (do not edit)` banner (the cove analogue of flue's `// Auto-generated by flue`), plus the citation header on the *generator* source.
- **[08 §3 Execution boundary](../../design/08-conventions-and-execution-boundary.md#3-the-execution-boundary-core-philosophy)** — generated wiring is pure orchestration glue: it registers handlers and routes, and **never** touches a box, the LLM, or sandbox FS. No `"use node"` directive belongs in the generated registry/router files (those run as queries/mutations + the `httpRouter`); the box is only reached from the P2/P4 engine actions.
- **[D13 skills resolve at the call site](../../design/08-conventions-and-execution-boundary.md#skills-resolve-at-the-call-site-not-in-the-sandbox)** — the m3 packaging seam reads `SKILL.md` from the **host/repo at build time** to produce catalog-import rows; it must NOT emit any code that reads `SKILL.md` from a sandbox FS at runtime. Packaging is the *authoring* side; runtime resolution stays catalog-table.
- **[D18 distinct run kind](../../design/07-risks-and-decisions.md)** — the workflow-route codegen wires `POST /workflows/:name` such that a workflow invoke is observable as a `kind:'workflow'` run (the discriminator + route handler are defined in P8; codegen only binds the declared handler to the route — it must not conflate workflow names into the agent registry).

## Acceptance

Start from [06 P8.5's bar](../../design/06-phase-roadmap.md#-phase-85--cli--codegen):

- [ ] **`cove dev` boots a Convex deployment with a declared `defineAgentRegistry`.** Run `cove dev` against the smoke fixture; it runs codegen + validation, then `convex dev` starts and the declared agent is addressable (a `POST /agents/<name>/<id>` from P8 reaches the registered `CreatedAgent`).
- [ ] **Codegen produces a wired `convex/http.ts` entry.** After `cove build`, the generated `convex/http.ts` mounts the P8 agent + run routes and imports the generated agent resolver; `tsc --noEmit` passes on the emitted output.
- [ ] **Declared `defineWorkflow` handlers wire to their `POST /workflows/:name` routes.** The generated workflow binding maps the fixture's `defineWorkflow` to `POST /workflows/<name>`; a `client.workflows.invoke(name, input)` / `curl POST /workflows/<name>` reaches the handler and surfaces as a **`kind:'workflow'`** run (distinct from agent runs, [D18](../../design/07-risks-and-decisions.md)).
- [ ] **An invalid registry fails validation with a `[cove]` diagnostic *before deploy*.** Each negative fixture (missing/unresolvable provider; undeclared subagent; duplicate agent name; a value lacking `__coveCreatedAgent`; an invalid name vs `^[A-Za-z][A-Za-z0-9_-]*$`) makes `cove build`/`cove deploy` exit non-zero with a single `[cove] …` line and **no** `convex deploy` is spawned.

Coverage-audit additions:

- [ ] **App-bound boundary held** — assert the generated + authored registry/workflow files are NOT exported from [`src/runtime/index.ts`](../../../src/runtime/index.ts) (grep the barrel: no `defineAgentRegistry`/`defineWorkflow`/`agentRegistry` re-export). The runtime barrel stays V8-safe/Convex-free.
- [ ] **Idempotent codegen** — running `cove build` twice with no source change rewrites **zero** generated files (the `changed` tracker), so `convex dev`'s watcher does not churn.
- [ ] **User-authored `convex/http.ts`/`app.ts` is preserved** — when present, codegen validates rather than overwrites it (mirrors flue's app.ts-wins branch).
- [ ] **`cove add` is a deferred stub** — invoking it prints the `[cove] … deferred (m7)` notice and exits 0/2 without scaffolding (m7 non-goal honored).
- [ ] **m3 packaging is opt-in + host-sourced** — with `config.skills` set, `cove build` parses a host `SKILL.md` via `parseSkillMarkdown` and emits a catalog-import payload; with it unset, no skill packaging runs. No `with { type: 'skill' }` import appears in any emitted file (assert by grep).
- [ ] **No brand leakage** — grep the generated artifacts for `flue`/`__flueCreatedAgent`/`[flue]`/`pi`-as-identifier → zero hits.

## Risks & gotchas

- **Convex owns dev/deploy — don't re-implement a server.** flue's [build-plugin-node.ts](../../../../flue/packages/cli/src/lib/build-plugin-node.ts) emits a full Hono `serve()` + coordinator + sqlite + IPC entry; **none** of that ports. The cove "build" emits *registration glue* (`httpRouter` wiring + a name→handler resolver) and shells out to `convex dev`/`convex deploy`. Resist porting the Vite bundle path — it has no Convex analogue ([08 §5](../../design/08-conventions-and-execution-boundary.md#5-dropped--obsolete-explicit)).
- **Registry must NOT leak onto the `@cove/runtime` barrel.** The most likely accidental regression: re-exporting `defineAgentRegistry`/`defineWorkflow` from [`src/runtime/index.ts`](../../../src/runtime/index.ts) "for convenience." That breaks the V8-safe/Convex-free barrel invariant ([08 §1](../../design/08-conventions-and-execution-boundary.md#1-naming--namespace), [05](../../design/05-public-api-and-sdk.md#agent-registry-defineagentregistry)). They stay app-bound under `convex/`. Add the grep guard to the acceptance suite.
- **Generated files run as Convex functions, not `"use node"` actions.** The emitted `convex/http.ts` + resolver are the orchestration/persistence layer — they must stay box-free and must not carry a `"use node"` directive ([08 §3](../../design/08-conventions-and-execution-boundary.md#which-convex-functions-may-touch-the-sandbox)). The box is reached only from P2/P4 engine actions.
- **Loading the user's registry for validation is the hard part.** `convex/agentRegistry.ts` imports Convex-app-bound code (and transitively the runtime). Loading it in-process for validation can drag in Convex globals. Prefer a sandboxed TS import (`tsx`/`jiti`) or AST-extract the default export; either way, wrap import failures in a `[cove]` diagnostic so a user typo is a one-line error, not a 50-line stack.
- **Validation now load-bearing — OS file-existence no longer guards names.** In flue, a bad agent name was caught by the filesystem (`agents/<name>.ts`). Here the registry **map keys** are the only gate, so `assertAgentName`/`assertUniqueNames`/the brand check are the *sole* defense against a malformed name reaching `setup` ([05](../../design/05-public-api-and-sdk.md#agent-registry-defineagentregistry)). Don't skip any of them.
- **Determinism / idempotent emission.** Codegen output must be byte-stable across runs (sorted map iteration, stable var-name indexing) so the `changed` tracker is meaningful and `convex dev`'s file watcher doesn't restart on no-op rebuilds (port flue's content-compare from [build.ts](../../../../flue/packages/cli/src/lib/build.ts) lines 215–232).
- **`cove deploy` must fail-closed.** Validation runs *before* `convex deploy` is spawned; a failed validation must never reach the Convex deploy step (no half-deployed app with an invalid registry). The negative acceptance test asserts no `convex deploy` child is spawned on validation failure.
- **m3 sensitive-file exclusion is a security boundary, not a nicety.** When porting the directory walk from [vite-import-attribute-plugin.ts](../../../../flue/packages/cli/src/lib/vite-import-attribute-plugin.ts), carry `SENSITIVE_DIRECTORIES`/`EXCLUDED_FILES`/`SENSITIVE_FILE_PATTERNS` verbatim so `.ssh`/`.aws`/`*.pem`/credentials never get packaged into a catalog payload.
- **Workflow vs agent run-kind conflation.** `defineWorkflow` codegen must keep workflows out of the agent registry/resolver — they are a **distinct run kind** ([D18](../../design/07-risks-and-decisions.md)). Emitting a workflow into the agent name map would re-introduce the agentName-rekey conflation P8 explicitly resolved with `runs.kind`.
