# Phase 6 — Harness facade + invoke + shell
> Wire the public `CoveContext.init → CoveHarness → CoveSession` facade onto Convex admission mutations + the durable workflow, port subagent/task delegation, the abort sequence, and the shell tool-event envelope. Design-of-record: [06 — Roadmap](../../design/06-phase-roadmap.md) + [05 — Public API](../../design/05-public-api-and-sdk.md), [04 — Durable Engine](../../design/04-durable-engine.md), [08 — Conventions](../../design/08-conventions-and-execution-boundary.md). Decisions: [D1–D19](../../design/07-risks-and-decisions.md).

## Goal & scope

Phase 6 makes a prompt runnable from user code. It builds the **admission layer** (`convex/invoke/`) and the **public facade** that sits on top of it. Concretely:

- `convex/invoke/` mutations: `submitPrompt`, `submitTask`, `stopActive` — the Convex entry points that admit a submission (insert an `agentRequests` row), kick the durable workflow (`workflow.start(agentRun, {requestId})`), and cancel it.
- The `CoveContext.init(agent, options) → CoveHarness` facade and the `CoveHarness → CoveSession` chain whose `prompt` / `skill` / `task` / `shell` / `fs` map onto those mutations + the workflow's terminal `agentRequests.status`.
- **Subagent / task delegation** — `createTaskTool` spawning a **nested `agentRun` workflow** under reserved `task:*` sessions, with `taskDepth` / `MAX_TASK_DEPTH` and declaration guards (`TaskDepthExceededError`, `SubagentNotDeclaredError`). Cascade-delete over `taskSessions` is the P5 hook this phase consumes.
- **`abort.ts` mapping** — port `createCallHandle` / `composeTimeoutSignal` / `abortErrorFor` and wire the atomic abort sequence `session.abort() → invoke.stopActive → one mutation (workflow.cancel + status=cancelled)`.
- **PORT `src/runtime/shell.ts`** — `execShellWithEvents` + `redactEnvValues` (keys-only redaction, §4.11 security) + `getErrorMessage`, shared by `session.shell()` (with transcript triple) and `harness.shell()` (no transcript).
- The named **admission-contract test (M6)** asserting the five surviving ADMISSION invariants on the Convex admission path.

**Out of scope (deferred):** HITL (`step.waitForEvent`, P7), HTTP/auth transport + `authorize` hook (P8), reactive event subscription + native SDK iterator (P9), skills catalog + `session.skill()` routing internals + MCP (P10), compaction summarization step (P12). `session.compact()` and `session.skill()` **method surfaces** are stubbed here (they admit a `compact`/`skill` request) but their loop behavior lands in P10/P12.

## Hardened-contract obligations

- **[08 §3](../../design/08-conventions-and-execution-boundary.md#3-the-execution-boundary-core-philosophy) — execution boundary.** `invoke/*` are **mutations** and must **never touch the box** (admission + scheduling + cancel only). `harness.shell()` / `session.shell()` *do* touch the box via `env.exec`, but the facade runs in user/Node space outside the workflow — it is the host-issued shell path, distinct from model-issued `dispatchTools`. The mutation that admits a shell-driven request still resolves the box lazily only inside the engine action, never in `invoke/*`.
- **[08 §4.3](../../design/08-conventions-and-execution-boundary.md#43-cancel-short-circuit) — cancel short-circuit (atomic).** `stopActive` is **one** mutation doing `workflow.cancel(convexWorkflowId)` **and** `agentRequests.status = "cancelled"` together — never two round-trips. `session.abort()` → `invoke.stopActive` is the only abort path. (The `dispatchTools` per-tool re-check that *discards late results* is P4's half; this phase guarantees the atomic flip that P4 reads.)
- **[08 §4.11](../../design/08-conventions-and-execution-boundary.md#411-shell-tool-event-envelope--env-redaction) — shell envelope & env redaction (SECURITY).** `execShellWithEvents` emits a `tool_start` + terminal `tool` pair with a **shared `toolCallId`**, `toolName:'bash'`, `durationMs`. `ShellOptions.env` values are **keys-only `<redacted>`** in **every** event **and** in the persisted `sessionEntries` triple; only `env.exec` receives real values. Failure branch records `details:{command, exitCode:-1}` via `getErrorMessage`. `session.shell()` appends the **redacted** transcript triple (`appendShellTriple`); `harness.shell()` appends **none**.
- **Subagent guards (D-level, [04 §Subagents](../../design/04-durable-engine.md)).** Each nested spawn increments `taskDepth`; `taskDepth >= MAX_TASK_DEPTH` throws `TaskDepthExceededError`; requesting an undeclared subagent throws `SubagentNotDeclaredError`. Reserved `task:*` names rejected for user sessions (`assertPublicSessionName`).
- **ADMISSION invariants ([08 §5](../../design/08-conventions-and-execution-boundary.md#5-dropped--obsolete-explicit), [D5](../../design/07-risks-and-decisions.md)).** Asserted here (the only survivors of the dropped `AgentSubmissionStore`): replay-idempotent admission; same-id-different-payload → conflict; concurrent same-session prompt → serialize/supersede via `by_session_and_status`; settled receipt retained after session delete; `deleteSession` blocks new admission.

## Acceptance

Start from [06 P6's bar](../../design/06-phase-roadmap.md#-phase-6--harness-facade--invoke) plus the coverage-audit additions:

1. **Facade happy path.** `const h = await ctx.init(agent); const s = await h.session(); await s.prompt("hi")` resolves a `PromptResponse` (status reaches `completed`, `finalText` populated). Driven against a stubbed/mock engine so no live provider is needed.
2. **Serialize / supersede.** A second concurrent `s.prompt(...)` on the **same** session either rejects with `SessionBusyError` (serialize) **or** supersedes the first (prior head → `status="cancelled"`, `cancelReason="superseded"`, its workflow cancelled) per flue semantics — and exactly one head is `running` afterward (`by_session_and_status` returns a single active row).
3. **Task delegation.** `s.task("...", {agent:"researcher"})` admits a `kind:"task"` request under `task:<parent>:<taskId>`, appends the parent's `TaskSessionRef`, starts a **nested** `agentRun`, and the child's final answer returns into the parent.
4. **Depth + declaration guards.** A delegation past `MAX_TASK_DEPTH` throws `TaskDepthExceededError`; `s.task("...", {agent:"undeclared"})` throws `SubagentNotDeclaredError`; `h.sessions.create("task:foo")` throws (reserved name).
5. **Abort.** `s.abort()` calls `invoke.stopActive` → one mutation cancels the workflow + flips `status="cancelled"` (`cancelReason:"stop"`); the awaiting `CallHandle` rejects with an `AbortError`; a late `appendToolResult` for that request is discarded (status-gated).
6. **Shell envelope — `session.shell()`.** Emits a `tool_start` + `tool` pair (`toolName:'bash'`, shared `toolCallId`, `durationMs`); `env` keys appear but **every value is `<redacted>`** in both events; the redacted transcript triple is appended to `sessionEntries`; the **real** env values are observed by the `env.exec` mock. A failing command records `details:{command, exitCode:-1}` and a message via `getErrorMessage`.
7. **Shell envelope — `harness.shell()`.** Emits the **same** event pair with redacted env, but appends **no** transcript (no session).
8. **Named admission-contract test (M6)** — all five pass:
   - **Replay-idempotent:** `admitDispatch`/`submitPrompt` with the **same** `submissionId` + payload returns the same `requestId` and inserts **one** row.
   - **Payload conflict:** the **same** `submissionId` with a **different** payload raises a conflict (no second row, no silent overwrite).
   - **Head selection:** a 2nd concurrent same-session prompt selects the head via serialize/supersede (`by_session_and_status` yields exactly one active head).
   - **Receipt retained:** a settled session's request **receipt survives** after `sessions.delete` (the terminal `agentRequests` row / its retained receipt is not cascaded away with the entries).
   - **Delete blocks admission:** while `sessions.state === "deleting"`, a new `submitPrompt` throws `SessionDeletedError`.
9. **`tsc --noEmit` exits 0.**

## Risks & gotchas

- **`formatBashResult` import boundary.** flue's `shell.ts` imports `formatBashResult` from `agent.ts`, which pulls in TypeBox + the whole tool factory. **Do not** drag `agent.ts` onto the pure runtime barrel just for one formatter — port `formatBashResult` (+ its `truncateTail`/`MAX_READ_LINES`/`MAX_READ_BYTES` constants) into a tiny `src/runtime/bash-format.ts` and import from there. Keep `shell.ts` dependency-light.
- **No box / no workflow in `invoke/*` mutations.** These are **mutations** — they cannot `step.run*`, cannot await an action, cannot touch `@upstash/box`. They only insert/patch rows and call `workflow.start`/`workflow.cancel` on the component. Anything that needs the box (the shell envelope's `env.exec`) runs in the **facade/Node** layer or the **engine action**, never here.
- **`stopActive` atomicity is load-bearing (§4.3).** `workflow.cancel` + the `status="cancelled"` patch must be in the **same mutation** so a replay never observes a cancelled workflow with a still-`running` row (which would let `dispatchTools` keep executing). Do not split into two mutations or a mutation + scheduler.
- **Serialize vs supersede is a policy, not a default-both.** flue's `runExclusive` throws `SessionBusyError` on a concurrent op (serialize). The "supersede" path is the SDK-level latest-wins used by channels/HTTP. Implement **serialize as the default** and supersede as an explicit opt (or per configured policy) — and make the M6 head-selection assertion test whichever you ship, not both ambiguously.
- **Client-side gate ≠ server-side gate.** The facade's `runOperation`/`activeOperation` is an in-process convenience for a single client; the **authoritative** serialization is the server `by_session_and_status` head selection. Two *different* clients hitting the same session only collide at the server gate — the test must exercise the **server** path (via `submitPrompt`), not just the client guard.
- **Nested workflow ≠ in-process recursion.** `task` must `workflow.start` a **separate** `agentRun` (its own journal, independent recovery), not call into the parent action. The parent step **awaits** the child's terminal status. A crashed child recovers on its own journal; the parent's await resumes from the journaled checkpoint.
- **`taskSessions` append must precede the child start** so a crash between create-and-start still leaves the cascade-delete link (no orphaned `task:*` session). Make the parent-row patch + admit a single mutation where possible.
- **Redaction happens before the record hook.** In `execShellWithEvents`, `args.env = redactEnvValues(options.env)` is computed **once**, up front, and the **same redacted `args`** flows into both the events *and* the `record` (transcript) hook — while `env.exec` receives `options.env` (real). Never re-derive args for the transcript from the raw options, or a secret leaks into a row. The M6/shell test must assert the row contains `<redacted>` and the exec-mock saw the real value.
- **Replay determinism of admission.** Admission keyed on `submissionId` must be a pure CAS-style insert (query `by_submission`, then insert) so a re-submitted dispatch is idempotent. Generate `submissionId` **on the caller side** (or deterministically) — a server-side `crypto.randomUUID()` per call would break replay-idempotency. Mirror flue's dispatch-id semantics.
- **`CallHandle` unhandled-rejection trap.** When porting `abort.ts`, keep the `promise.catch(() => {})` guard — fire-and-forget `prompt()`/`abort()` calls must not surface an unhandled rejection while still rejecting for callers that do `await`.
