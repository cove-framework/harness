# Phase 9 — Reactive events + native SDK + @cove/react
> Stand up the durable+reactive `events` write/read path (the `observe()` substitute), the `createCoveClient` native SDK over `ConvexReactClient` (`runs.events()` async-iterates a reactive query), and `@cove/react` hooks wrapping the ported agent-reducer `UIMessage[]` view-model. Design-of-record: [06 — Roadmap](../design/06-phase-roadmap.md) + [05 — Public API & SDK](../design/05-public-api-and-sdk.md), [03 — Data Model](../design/03-data-model-sor.md), [08 — Conventions & Execution Boundary](../design/08-conventions-and-execution-boundary.md). Decisions: [D1–D19](../design/07-risks-and-decisions.md).

## Goal & scope

Phase 9 replaces flue's Durable-Streams/SSE consumer transport with **Convex reactivity**. Three deliverable groups, all consumer-side (no LLM loop, no box):

1. **`convex/events/`** — the write path (append a decorated `CoveEvent` row with a monotonic per-`streamKey` `seq`, batched), the read path (a **reactive** `listForStream`/`listForSubmission` query that *is* the stream), and the image-redaction guard so events never carry raw bytes (§4.8). This is the durable `observe()` substitute referenced throughout 03/04.
2. **`src/sdk/`** — `createCoveClient({ convex })` yielding a `CoveClient` whose `agents.send`/`agents.prompt`, `runs.get`, and `runs.events(handle.requestId)` return the **same `CoveEvent`/`RunRecord` types** as flue. `runs.events()` async-iterates a `CoveEventStream` fed by a reactive query (catch-up by `seq` + live tailing), **not** an SSE reader. On the wire `CoveApiError` replaces `FlueApiError`.
3. **`src/react/`** + **`@cove/react`** — the **ported `agent-reducer`** that folds the reactive event stream into a `UIMessage[]` view-model with optimistic-send correlation (`pendingSends`/`activeSubmissionIds`/`settledSubmissionIds`/`recentEventIds`) and `IMAGE_DATA_OMITTED` file-part reuse, reconciled against **PATCHED snapshots** (§4.6); plus the hooks `useCoveClient`/`useAgentPrompt`/`useRunEvents`/`useCoveRun` bound to `useQuery` subscriptions.

**In scope:** the event substrate, the native client, the reducer + hooks, and the deterministic reducer unit tests (the port of flue's `agent-reducer.test.ts`). **Out of scope (deferred):** the workflow-run SDK/hook surface (`client.workflows.invoke`, `useCoveWorkflow`) — that lands with P8/P8.5 where `kind:'workflow'` runs and the `POST /workflows/:name` route exist; the SDK `CoveClient` interface carries a `workflows` *type slot* so P8 fills it without a breaking change, but P9 ships the `agents`/`runs` surfaces only. The HTTP poll/submit shim itself lands in P8; P9's SDK assumes the native `ConvexReactClient` path.

## Dependencies

- **P4 (Durable engine)** — must land first. The engine actions (`llmStep`/`dispatchTools`/`finalize`) are the **emitters** that call `events.append`; without them there is no event stream to read. P9 provides the append mutation + the redaction guard the engine writes through, but the engine must exist to drive it. The `agentRequestSteps` reactive read (the SSE-replacement substrate, 03) is co-consumed by `useRunEvents` for delta-flush re-render.
- **P6 (Harness facade + invoke)** — must land first. `invoke.submitPrompt` returns `{ sessionId, requestId, submissionId }` (05) — the ids the SDK `agents.send` and the `useAgentPrompt` hook hand back to subscribe with. The `operationId`/`turnId` correlation ids (03) are minted on the events write path that P6's operation envelope drives. The `submission_settled` recovery event the reducer consumes is emitted on the admission/recovery path.
- Pure prerequisites already on disk (P1): `CoveEvent`/`AttachedAgentEvent`/`Llm*`/`PromptUsage`/`AgentMessage` types in [`src/runtime/types.ts`](../../src/runtime/types.ts) and [`src/runtime/messages.ts`](../../src/runtime/messages.ts); the `events`/`agentRequestSteps` tables in [`convex/schema.ts`](../../convex/schema.ts).

P9 is a fan-out leaf off P6 (parallel to P7/P8) per the [critical path](../design/06-phase-roadmap.md#critical-path). It does not block P10/P12 but P12's E2E test subscribes through this SDK.

## Hardened-contract obligations

- **§4.6 Streaming commit & subscription semantics** — The reducer reconciles against **PATCHED snapshots, not an append log**: `useRunEvents` re-derives `UIMessage[]` on each delta flush, and `text_delta`/`thinking_delta` fold into the *current* assistant part in place (the reducer's `reduceTextDelta`/`reduceThinkingDelta` append to `parts.at(-1)` when `state !== 'done'`). Because deltas coalesce into the step row and the reactive query re-emits the patched row, the consumer must be **idempotent on re-delivery** — `streamEventId` dedup (`recentEventIds`) + `message_end` terminal reconciliation (`snapshotMessage` replacing provisional parts by `turn:${turnId}` id) guarantee a re-emitted batch produces the same `messages` (this is exactly flue's "interrupted partial batch is replayed" test). The SDK `CoveEventStream` cursor advances **only after a batch is fully yielded** (at-least-once), so a re-emit never skips an undelivered event.
- **§4.8 Image pipeline (event path)** — `events` rows **never** carry raw image bytes: `convex/events/append.ts` runs `redactEventImages` before insert, replacing every image block's `data` with `IMAGE_DATA_OMITTED` while keeping `mimeType`. On the consumer side the reducer's `snapshotMessage` **reuses the prior file part** when `block.data === IMAGE_DATA_OMITTED && prior?.mediaType === block.mimeType` (the m1 reuse), so an omitted redelivered image does not flicker or re-upload. `imageUrl` returns the sentinel verbatim for an omitted block.
- **§3 Execution boundary** — P9 is entirely **query/mutation + client** code: `convex/events/*` are queries (read) and an internal mutation (append) — **never** a `"use node"` action, never touching the box or the LLM. The SDK/react packages are pure consumers. The `append` mutation is the only write, called *by* the engine actions (P4); P9 does not add an emitter.
- **03 grouping ids** — `operationId` (1:1 with `submissionId`) and `turnId` (per LLM turn) are **opaque correlation ids on the event payload**, not table keys, and `turnId ≠ (requestId, stepNumber)`. The reducer groups assistant messages by `turn:${turnId}` (its message id) and correlates user echoes by `submission:${submissionId}:user:0` — it must read these off `event.turnId`/`event.submissionId`/`event.dispatchId`, exactly as flue does. `listForSubmission` filters by `submissionId` via `by_submission`.
- **§1 Naming** — every new identifier is `Cove*` (`CoveClient`/`createCoveClient`/`CoveApiError`/`CoveEventStream`/`CoveStreamOptions`/`CoveProvider`/`useCoveClient`); the `[cove]` prefix on every thrown error and `cove_error` codes; `observe`/`events`/`runs`/`session` stay (generic, un-branded). Every new file opens with the reference-header (§2).

## Acceptance

Start from [06's P9 bar](../design/06-phase-roadmap.md#-phase-9--reactive-events--native-sdk) plus the coverage-audit additions:

1. **Full live event sequence, no SSE.** A client (via `createCoveClient`) subscribes to a run and observes the full ordered sequence `run_start … text_delta … tool … run_end` (for agent runs: `operation_start … message_start … text_delta … tool_start … tool … turn … idle`) live through the reactive query — driven by `convex.onUpdate`, with **no Durable-Streams / SSE** code in the path.
2. **`@cove/react` renders the same stream reactively.** A `useRunEvents(requestId)` hook renders a `UIMessage[]` that updates on each batched delta flush; the assistant message's text part grows in place across `text_delta` events and finalizes (`state:'done'`) on `message_end`.
3. **Optimistic-send reconciles with no duplicate** (the named reducer tests pass against cove types): receipt-before-echo and echo-before-receipt both collapse to a single `submission:${submissionId}:user:0` message; `local_send_failed` removes the optimistic message and sets `status:'error'`; a second pending submission stays `submitted` when the first goes `idle`. (Ports flue's `agent-reducer.test.ts` "reconciles receipt-before-echo", "reconciles echo-before-receipt by dropping the optimistic duplicate", "keeps another local submission pending".)
4. **Redacted image reuses the prior file part** (m1): given a prior `UIMessage` with a `file` part of `mediaType X`, a re-emitted `message_end` whose image block carries `data === IMAGE_DATA_OMITTED && mimeType === X` reuses the prior `file` part rather than producing a `data:` URL of the sentinel — no flicker, no re-upload.
5. **Events carry no bytes** (§4.8): an `append` of an event with a base64 image block persists an `events` row whose image `data` is `IMAGE_DATA_OMITTED` and whose `mimeType` is intact; the raw base64 never lands in the row.
6. **Replay-idempotent consumer** (§4.6): replaying the same partial event batch through `reduceAgentEvent` produces identical `messages` (ports flue's "does not duplicate provisional parts when an interrupted partial batch is replayed"); the `CoveEventStream` never re-yields an already-delivered `seq`.
7. **Green gates:** `tsc --noEmit` exits 0; `vitest run` passes (reducer + redaction + event-stream suites).

## Risks & gotchas

- **Convex `onUpdate` re-delivers the whole result set.** A reactive query subscription fires with the *entire* current query result on every tick, not a delta. The `CoveEventStream` and `AgentStore` must diff against `currentSeq`/`recentEventIds` and yield/dispatch only new rows, or every flush re-yields the whole history. The reducer's `streamEventId` dedup is the second line of defense; the stream cursor (`sinceSeq`) is the first.
- **PATCHED step rows, not append-only (§4.6).** The `agentRequestSteps.text`/`reasoning` channels are *patched in place* ~10–20×/turn. If `useRunEvents` co-subscribes the step row, the same step re-emits with growing text — the reducer must treat that as "replace the current streaming part", not "append a new delta". (The `events` `text_delta` rows *are* append-only; the step-row text is cumulative — do not mix the two semantics.)
- **Query is reactive, not a node action — keep it box-free and pure.** `convex/events/*` must be queries + an internal mutation only; no `"use node"`, no provider/box imports. A stray `node`-only import would break the V8 query runtime and pull the box into the read path (§3 violation).
- **`AgentMessage` has a `'signal'` role flue's `LlmMessage` lacks.** The ported reducer's `message_*` handlers and `snapshotMessage` must early-return on `role:'signal'` (and keep the `toolResult` guard) — otherwise a signal message lands as a malformed `UIMessage`.
- **`RunRecord`/`RunStatus` shape mismatch.** Do **not** copy flue's `status:'active'|'completed'|'errored'` workflow-only `RunRecord`. Cove's `runs` row uses `running|completed|failed|cancelled` and carries `kind` + `agentName` (03/schema). Mirroring flue's shape would mistype the SDK against the real table.
- **No `streamUrl`/`offset` anymore.** `AgentSendResult` is `{ sessionId, requestId, submissionId }` (05) — every consumer that read `streamUrl`/`offset` in flue must switch to subscribing by id. Leaving the old fields in the type is a silent foot-gun.
- **Monotonic `seq` under concurrency.** `nextSeq` does a read-then-+1; it is safe only because writes to one `streamKey` are serialized by the per-session gate (P6) and the engine's single-writer loop. If a future emitter writes the same stream concurrently, `seq` could collide — keep `append` the single writer per stream.
- **Workflow surface is deferred — type-slot only.** Shipping a live `client.workflows.invoke`/`useCoveWorkflow` in P9 would dangle on `kind:'workflow'` runs and the `POST /workflows/:name` route that don't exist until P8/P8.5. Leave a typed-but-throwing slot so P8 fills it without an API break; do not port `workflow-run.ts`/`use-workflow.ts` here.
- **React peer dep, not a hard dep.** `@cove/react` must declare `react` as a *peer* dependency (matching flue) so consumers' single React instance is used; bundling React would break hooks. The runtime/sdk barrels stay React-free.
