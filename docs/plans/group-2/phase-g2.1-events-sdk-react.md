# G2.1 — Reactive events substrate + native SDK + `@cove/react`

> *Completes [P9](../group-1/phase-09-events-sdk.md). Group-1 P9 landed the **types and the read-side stand-ins** — the `events` table ([`schema.ts:320`](../../../convex/schema.ts)), the `CoveEvent`/`CoveEventInput`/`Llm*` contract ([`types.ts:804`](../../../src/runtime/types.ts)), and the reactive `steps.listForRequest` / `requests.get` queries ([`steps.ts`](../../../convex/steps.ts), [`requests.ts`](../../../convex/requests.ts)) — but **the engine emits zero events** and the only SDK is the poll-transport facade in [`src/sdk/index.ts`](../../../src/sdk/index.ts). G2.1 finishes P9 end-to-end: it builds the `convex/events/*` write+read path, **wires the emitter call sites into the engine** (the largest hidden cost — the read path is empty until they land), reconciles the SDK into the real reactive consumer client, and ships `@cove/react` (the ported `agent-reducer` + hooks). After this phase a `@cove/react` consumer renders a live agent run as a growing `UIMessage[]` over Convex reactivity with **zero SSE**.*

## Goal & scope

P9's group-1 cut stopped at the *consumer types* and two reactive stand-in reads (`steps`/`requests`). What is **missing and in-scope here**:

1. **`convex/events/`** — the durable `observe()` substitute. The write path (`append`: decorate → redact → monotonic `seq` → fan out to stream keys), the per-stream `seq` helper, the full-`CoveEvent` `redactEventImages`, and the reactive `listForStream`/`listForSubmission` queries that *are* the stream.
2. **Engine emitter wiring** — **the hidden cost.** The engine ([`setup.ts`](../../../convex/engine/setup.ts), [`llmStep.ts`](../../../convex/engine/llmStep.ts) → [`decode.ts`](../../../convex/engine/decode.ts), [`dispatchTools.ts`](../../../convex/engine/dispatchTools.ts), [`finalize.ts`](../../../convex/engine/finalize.ts), [`runHandler.ts`](../../../convex/engine/runHandler.ts)) currently makes **no** `events.append` call — `grep -rn "events.append" convex/` returns nothing. The reactive read path is empty until the emitter call sites land. This is in-scope, not just the `append` mutation.
3. **`convex/runs.ts`** — `get`/`listRuns` so the SDK `runs.get(runId)` has a backend.
4. **`src/sdk/{types,client,event-stream}.ts`** — reconcile/replace the poll shim with the reactive consumer client whose `runs.events()` async-iterates `convex.onUpdate(api.events.listForStream)` with catch-up-by-`seq` + live tail + cancel.
5. **`src/react/*`** + `@cove/react` — the ported reducer (`role:'signal'` early-return + `IMAGE_DATA_OMITTED` file-part reuse) + `CoveProvider`/`useAgentPrompt`/`useRunEvents`/`useCoveRun`.
6. **Tooling** — react/test devDeps, `react>=18` peer, package `exports`, tsconfig `jsx`, a vitest happy-dom project; and the deterministic reducer + event-stream + redaction tests.

**Non-goals / scope cuts (deferred):**

- **Workflow SDK/hook surface.** `client.workflows.invoke` / `useCoveWorkflow` stay a **typed-but-throwing slot** (D18). The `runs` table has no `kind` discriminator yet ([`schema.ts:295`](../../../convex/schema.ts) is `status` only); workflow runs and `POST /workflows/:name` land in **[G2.4](phase-g2.4-cli-codegen.md)**. Do not port flue's `workflow-run.ts`/`use-workflow.ts`.
- **HTTP/DS transport.** The DS reconnect/offset machinery and `@durable-streams/client` are dropped ([08 §5](../../design/08-conventions-and-execution-boundary.md#5-dropped--obsolete-explicit)); the HTTP submit/poll shim already exists in [`convex/http.ts`](../../../convex/http.ts).
- **`runs`-table row writer.** `convex/runs.ts` *reads* the `runs` table; nothing in the engine writes it today (the run-row lifecycle writer is **[G2.4](phase-g2.4-cli-codegen.md)**/D18 scope). `runs.get` therefore falls back to the `agentRequests`-backed point-in-time view (parity with the existing [`requests.get`](../../../convex/requests.ts) and the `GET /runs/:runId` route in [`http.ts:105`](../../../convex/http.ts)) and returns `null` when no `runs` row exists. See Risks.
- **OTel observer.** The read-side fold over `events` is **[G2.5](phase-g2.5-glue-sample-otel.md)**.
- **`convex-test` event-stream store harness.** **[G2.6](phase-g2.6-convex-test-suite.md)**; G2.1 ships in-process reducer/stream tests against fakes, not `convex-test`.

## Hardened-contract obligations

- **[§4.6 Streaming commit & subscription semantics](../../design/08-conventions-and-execution-boundary.md#46-streaming-commit--subscription-semantics)** — The reducer reconciles against **PATCHED snapshots, not an append log.** `useRunEvents` re-derives `UIMessage[]` on each delta flush; `text_delta`/`thinking_delta` fold into the current assistant part in place (`reduceTextDelta`/`reduceThinkingDelta` append to `parts.at(-1)` when `state !== 'done'`, reducer lines 202-248). Because the delta-batcher coalesces token deltas ([`deltaBatcher.ts`](../../../convex/engine/deltaBatcher.ts) emits *deltas since the last flush*) and `patchStreaming` ([`steps.ts:71`](../../../convex/engine/steps.ts): `text: row.text + delta`) **appends** them onto the step row so the *row's* `text` is cumulative, and the reactive query re-emits the patched row, the consumer must be **idempotent on re-delivery** — `streamEventId` dedup (`recentEventIds`) + `message_end` terminal reconciliation (`snapshotMessage` replacing provisional parts by `turn:${turnId}`) guarantee a re-emitted batch yields the same `messages` (flue's "interrupted partial batch is replayed" test). The `CoveEventStream` cursor advances **only after a batch is fully yielded** (at-least-once), so a re-emit never skips an undelivered `seq`. **Two distinct text semantics must not be mixed:** the `events` `text_delta` rows are append-only deltas; the `agentRequestSteps.text` channel is cumulative (patched ~10-20×/turn). If `useRunEvents` co-subscribes the step row, treat its text as "replace the current streaming part", never "append a new delta".
- **[§4.8 Image pipeline](../../design/08-conventions-and-execution-boundary.md#48-image-pipeline)** — `events` rows **never** carry raw image bytes. `convex/events/append.ts` runs `redactEventImages` before insert, replacing every image block's `data` with `IMAGE_DATA_OMITTED` while keeping `mimeType`. On the consumer side `snapshotMessage` **reuses the prior `file` part** when `block.data === IMAGE_DATA_OMITTED && prior?.mediaType === block.mimeType` (the m1 reuse, reducer lines 397-404), so an omitted redelivered image does not flicker or re-upload; `imageUrl` returns the sentinel verbatim for an omitted block.
- **[§3 Execution boundary](../../design/08-conventions-and-execution-boundary.md#3-the-execution-boundary-core-philosophy)** — G2.1 is entirely **query/mutation + client** code. `convex/events/*` are queries (read) + one internal mutation (append) — **never** `"use node"`, never the box or the LLM (the §3 table lists `finalize` as a mutation and the persistence layer as box-free). The emitter call sites land *inside the existing engine actions/mutations* (`setup`/`decode`(via `llmStep`)/`dispatchTools`/`finalize`) — they call `internal.events.append`, they do not become node-capable. The SDK/react packages are pure consumers; `src/react` imports nothing from `convex/` (hence the `IMAGE_DATA_OMITTED` re-home).
- **[03 grouping ids]** — `operationId` (1:1 with `submissionId`) and `turnId` (per LLM turn) are **opaque correlation ids on the event payload**, not table keys; `turnId ≠ (requestId, stepNumber)`. The reducer groups assistant messages by `turn:${turnId}` (`messageId`, line 433) and correlates user echoes by `submission:${submissionId}:user:0` (`userMessageId`, line 439), reading them off `event.turnId`/`event.submissionId`/`event.dispatchId` exactly as flue does. `listForSubmission` filters by `submissionId` via the `by_submission` index ([`schema.ts:333`](../../../convex/schema.ts)).
- **[§1 Naming](../../design/08-conventions-and-execution-boundary.md#1-naming--namespace) + [§2 reference-header](../../design/08-conventions-and-execution-boundary.md#2-reference-header-convention)** — every new identifier is `Cove*` (`CoveEventStream`/`CoveStreamOptions`/`CoveProvider`/`CoveApiError`/`useCoveClient`); `[cove]` prefix on every thrown error; `observe`/`events`/`runs`/`session` stay (generic). **Every new file opens with the reference-header** citing flue/pi origin + npm package (e.g. `// Ported from flue · @flue/react · packages/react/src/agent-reducer.ts → @cove/react`).

## Acceptance

1. **Full live event sequence, zero SSE.** A `createCoveReactiveClient({ convex })` subscriber observes the full ordered agent sequence live through the reactive query — `operation_start → message_start → text_delta… → tool_start → tool → turn → idle` — driven by `convex.onUpdate`. **`grep -rn "durable-streams\|EventSource\|text/event-stream\|subscribeJson" src/sdk src/react` returns nothing** (no SSE/DS in the path).
2. **Emitter call sites land (the hidden cost is paid).** `grep -rn "internal.events.append\|emitEvent" convex/engine/{setup,llmStep,decode,dispatchTools,finalize}.ts` shows an emit at all four seams (it returns **0** today). An end-to-end agent run produces a non-empty `listForStream` result for its `instanceId`.
3. **Events carry no bytes** (§4.8). `append` of an event with a base64 image block persists an `events` row whose image `data === IMAGE_DATA_OMITTED` and whose `mimeType` is intact; the raw base64 never lands. Covered by the redaction unit test across **every** image-carrying variant.
4. **Monotonic `seq`, no re-yield.** `nextSeq` produces strictly increasing per-`streamKey` values; the `CoveEventStream` on a re-delivered whole-result tick **never re-yields an already-delivered `seq`** (event-stream test: a second identical `onUpdate` callback yields nothing new).
5. **`@cove/react` renders a growing `UIMessage[]`.** A `useRunEvents(requestId)` (happy-dom) renders a `UIMessage[]` whose assistant text part grows in place across `text_delta` and finalizes (`state:'done'`) on `message_end`.
6. **Optimistic reconcile collapses to one message** (the named reducer tests pass against cove types): **receipt-before-echo** and **echo-before-receipt** both collapse to a single `submission:${submissionId}:user:0`; `local_send_failed` removes the optimistic message + sets `status:'error'`; a second pending submission stays `submitted` when the first goes `idle`.
7. **Redacted image reuses the prior file part** (m1): given a prior `UIMessage` with a `file` part of `mediaType X`, a re-emitted `message_end` whose image block carries `data === IMAGE_DATA_OMITTED && mimeType === X` reuses the prior `file` part rather than producing a `data:` URL of the sentinel — no flicker.
8. **`role:'signal'` early-return guard.** A `message_*` event carrying a `role:'signal'` `AgentMessage` returns the reducer state unchanged (no malformed `UIMessage`) — a unit case beyond the 14 ported ones.
9. **Replay-idempotent consumer** (§4.6): replaying the same partial event batch through `reduceAgentEvent` yields identical `messages` (the ported "interrupted partial batch is replayed" case).
10. **Green gates:** `tsc --noEmit` exits 0 (incl. `.tsx` under `react-jsx`); `vitest run` passes (reducer 14 + signal-guard + redaction + image-reuse + event-stream), `src/react/**` under happy-dom; the **existing 28 test files still pass** (`sdk.test.ts` unbroken by the client reconciliation).

## Risks & gotchas

- **The engine emits zero events today — the empty-stream emitter cost is the bulk of the work.** `grep -rn "events.append" convex/` is empty. The deliverable is not the `append` mutation; it is the *call sites* threaded through `setup`/`decode`/`dispatchTools`/`finalize`. **Mitigation:** task 8 + the `emitEvent` helper centralize the fan-out; acceptance bar 2 greps for the seams.
- **Convex `onUpdate` re-delivers the whole result every tick.** A reactive subscription fires with the *entire* current query result on each reactive change, not a delta. **Mitigation:** the `CoveEventStream`/`AgentStore` diff against `currentSeq` (first line) and the reducer `streamEventId`/`recentEventIds` dedup (second line); yield only `seq > currentSeq`.
- **PATCHED step rows vs append-only events must not double-count text.** `agentRequestSteps.text`/`reasoning` are patched in place ~10-20×/turn — the delta-batcher ([`deltaBatcher.ts`](../../../convex/engine/deltaBatcher.ts)) emits *deltas since the last flush* and `patchStreaming` ([`steps.ts:71`](../../../convex/engine/steps.ts)) appends them, so the *row's* text is cumulative; the `events` `text_delta` rows, by contrast, stay append-only deltas. If `useRunEvents` co-subscribes the step row, treat its text as "replace the current streaming part", never "append a new delta". **Mitigation:** the reducer's `text_delta` handler appends; the step-row path (if used) must `snapshotMessage`-replace — keep the two semantics separate.
- **`redactEventImages` must cover every image-carrying variant.** Cove's `CoveEvent` union is wider than flue's coverage (`operation`/`task`/`turn.output` also carry results). A missed variant leaks bytes into a row. **Mitigation:** the redaction unit test exercises one fixture per image-bearing variant; copy-on-write returns input unchanged otherwise.
- **`AgentMessage` has a `role:'signal'` flue's `LlmMessage` lacks** ([`messages.ts:120`](../../../src/runtime/messages.ts)). The ported `message_*` handlers + `snapshotMessage` must early-return on `'signal'` (and keep the `toolResult` guard) — otherwise a signal message lands as a malformed `UIMessage`. **Mitigation:** acceptance bar 8 + the `Exclude<…, {role:'signal'}|{role:'toolResult'}>` type on `snapshotMessage`.
- **`createCoveClient` name collision.** `src/sdk/index.ts` already exports a `createCoveClient(client, refs)` — the runtime-facade transport that `sdk.test.ts` and `context.ts` depend on. The reactive consumer client is a *different* thing. **Mitigation:** task 12 keeps the existing export untouched and adds the reactive client under the distinct name `createCoveReactiveClient`; acceptance bar 10 keeps `sdk.test.ts` green.
- **`IMAGE_DATA_OMITTED` lives in `convex/`, the reducer is in `src/react/`.** The reducer must not import from `convex/` (boundary + V8). **Mitigation:** task 2 re-homes the constant to `src/runtime/event-image.ts`; `images.ts` re-exports.
- **`RunRecord`/`RunStatus` shape mismatch.** Do **not** copy flue's `status:'active'|'completed'|'errored'` workflow-only record. Cove's `runs` row is `running|completed|failed|cancelled` + `agentName`/`instanceId` ([`schema.ts:295`](../../../convex/schema.ts)). **Mitigation:** the source map pins the cove shape.
- **The `runs` table is unwritten.** `convex/runs.ts` reads `runs`, but no engine code inserts a `runs` row yet (the writer is G2.4/D18). **Mitigation:** `runs.get` falls back to the `agentRequests` point-in-time view (parity with `requests.get` + `GET /runs/:runId`) and returns `null` when absent; `listRuns` returns `[]` until the writer lands — documented, not silently broken.
- **Monotonic `seq` under concurrency.** `nextSeq` is read-then-+1; safe **only** because writes to one `streamKey` are serialized by the per-session gate (P6) + the engine's single-writer loop. A future concurrent emitter on the same stream could collide. **Mitigation:** keep `append` the single writer per stream; do not call it from two concurrent actions on the same `streamKey`.
- **React is a peerDep, not a hard dep.** `@cove/react` must declare `react` as a *peer* (matching flue) so the consumer's single React instance is used; bundling a second instance breaks hooks. **Mitigation:** task 1 sets `peerDependencies` + devDep-only install; the runtime/sdk barrels stay React-free.
- **Workflow surface stays a typed-but-throwing slot (D18).** Shipping a live `workflows.invoke`/`useCoveWorkflow` dangles on `kind:'workflow'` runs + `POST /workflows/:name` that don't exist until G2.4. **Mitigation:** typed slot that throws `[cove]`; no port of `workflow-run.ts`/`use-workflow.ts`.
