# G2.5 — Compaction auto-trigger + `session.skill()`/`compact()` facade + worked sample agent + OTel observer

> *Completes [P12](../group-1/phase-12-compaction-tests.md)'s glue + the post-P12 [Observability strand](../../design/06-phase-roadmap.md#observability-deferred). Group-1 P12 landed the **pure compaction half** ([`src/runtime/compaction.ts`](../../../src/runtime/compaction.ts): `deriveCompactionDefaults`/`estimateContextTokens`/`shouldCompact`/`prepareCompaction`) and the **explicit `compact` action** ([`convex/engine/compact.ts`](../../../convex/engine/compact.ts)) that summarizes + appends a `CompactionEntry` ([`store.ts:125`](../../../convex/sessions/store.ts) `appendCompactionEntry`). But the auto-trigger is **dead**: `shouldCompact` is hard-coded `false` in two places ([`decode.ts:189,199`](../../../convex/engine/decode.ts), noted at [`types.ts:94-95`](../../../convex/engine/types.ts)); the loop has no `compact` port ([`loop.ts:42`](../../../convex/engine/loop.ts)); `runHandler` wires no `compact` step ([`runHandler.ts:26`](../../../convex/engine/runHandler.ts)); `setup` neither derives compaction defaults nor honors `compaction:false` ([`setup.ts:92`](../../../convex/engine/setup.ts)). The author-facing facades throw: `session.compact()` is a `not_implemented` stub ([`context.ts:209-210`](../../../src/runtime/context.ts)), `session.skill()` is a `deferredCall` ([`context.ts:199-201`](../../../src/runtime/context.ts)). There is no sample agent and no OTel consumer anywhere. G2.5 finishes the P12 wiring (threshold auto-trigger as a journaled step), implements both facades through new transport seams onto the existing admission + catalog reads, ships the worked sample agent proving G1–G5, and ports flue's `@flue/opentelemetry` observer as a **read-side fold over the `events` table**.*

## Goal & scope

P12's group-1 cut stopped at the pure helpers and the manual `compact` action. What is **missing and in-scope here**:

1. **Threshold auto-trigger (the P12 glue).** Derive + **freeze** compaction settings at `setup` from model caps (honor profile `compaction:false`); replace the hard-coded `shouldCompact:false` in `decode.ts` with the real threshold computed from **persisted per-step usage**; add a **journaled `compact` step** to `RunLoopDeps` → `runAgentLoop` → `runHandler` (replay-safe via `step.runAction(internal.engine.compact.compact)`).
2. **Author facades.** Implement `session.compact()` and `session.skill()` in [`context.ts`](../../../src/runtime/context.ts) through two new `CoveTransport` seams — `submitCompact` (onto a `compact`-kind admission) and `submitSkill` (catalog `getSkill` → submit the skill body as a prompt → `PromptResponse`). **Catalog-only, no box** ([08 §3](../../design/08-conventions-and-execution-boundary.md#skills-resolve-at-the-call-site-not-in-the-sandbox)).
3. **Worked sample agent** under `examples/agents/codeReviewer/` registered via `defineAgentRegistry`, demonstrating **G1–G5**: prompt + reactive streaming + a HITL-gated tool + a skill.
4. **OTel observer** — `convex/observability/` porting flue's [`opentelemetry/src/index.ts`](../../../../flue/packages/opentelemetry/src/index.ts) as a **read-side consumer of the `events` table** (G2.1's substrate), folding `CoveEvent` rows into run/turn/tool/compaction spans; promote `@opentelemetry/api` to a **direct dependency**.

**Non-goals / scope cuts (deferred):**

- **Overflow→retry mode.** Mode-2 (reactive overflow → compact → retry-**same-step**) is **explicitly out of this phase's auto-trigger work *except*** for landing the classifier so G2.6 can test it. Per [roadmap open Q6](README.md#open-questions-surface-before-committing) and [G2.6](phase-g2.6-convex-test-suite.md), the `isContextOverflow` classifier (`convex/engine/retry.ts`) + the same-`stepNumber` retry hook in `llmStep`/`runHandler` **land once here** (a thin addition next to the threshold path), but the *E2E proof* (overflow → retry → no double-charge) is **G2.6's** `tests/e2e/compaction.test.ts`. This phase ships threshold-mode + the overflow seam; it does not re-port the loop.
- **The three store contract harnesses + the multi-turn crash-recovery E2E + throughput gate.** Those are [G2.6](phase-g2.6-convex-test-suite.md) (gated on the `convex-test` offline install). G2.5's deterministic acceptance runs against `convex-test`/the `cove-test/mock` model where available and against in-process fakes otherwise.
- **A live OTel collector / exporter wiring.** The observer is the **span-mapping consumer**; standing up an `@opentelemetry/sdk-node` `NodeTracerProvider` + an OTLP exporter is an app-deployment concern, not framework scope. G2.5 ships the observer + an in-memory `Tracer` test; it does not bundle an exporter.
- **`session.task()` / `session.shell()` / `harness.fs`.** Still deferred stubs (P6 sandbox facade); untouched here.
- **New engine capabilities.** The loop is P4; this only adds the `compact` step + the threshold gate. No new channels, providers, or tools.

## Hardened-contract obligations

- **[§4.1 `llmStep` replay determinism *(critical)*](../../design/08-conventions-and-execution-boundary.md#41-llmstep-replay-determinism-critical).** The `compact` step is a journaled `step.runAction` checkpoint. On a mid-loop replay the workflow re-runs `runAgentLoop`, re-takes the `if (decision.shouldCompact)` branch (the decision is reconstructed from the finalized step row, [`decode.ts:184`](../../../convex/engine/decode.ts) `reconstructDecision`), and `step.runAction(compact)` returns the **cached** journal result — **no second `generateText` summarization call**. The compaction summary is byte-faithful across replay because it is journaled, not recomputed. This is acceptance bar 1's "mid-loop replay re-runs the journaled compact with NO second summarization call".
- **[§4.7 usage & cost](../../design/08-conventions-and-execution-boundary.md#47-usage--cost).** The in-loop threshold reads the **persisted per-step usage** (`agentRequestSteps.usage`, the widened rollup carrying `cacheRead`/`cacheWrite`/`cacheWrite1h` + `cost{}`, [`schema.ts:285`](../../../convex/schema.ts) / [`usage.ts`](../../../convex/engine/usage.ts)) via `estimateContextTokens` → `calculateContextTokens` ([`compaction.ts:55,108`](../../../src/runtime/compaction.ts)). It must **not** re-estimate from the chars/4 heuristic when a finalized usage row exists, or it fires inconsistently versus the manual `compact` action's `tokensBefore` (Risks).
- **[§3 execution boundary — skills resolve at the call site](../../design/08-conventions-and-execution-boundary.md#skills-resolve-at-the-call-site-not-in-the-sandbox).** `session.skill()` resolves the skill **only** from the `skills` catalog (`getSkill`, a Convex query) — **never** an `env.readFile`/sandbox FS walk. `submitSkill` provisions no box. Unknown skill → a typed `[cove]` error, not an FS miss.
- **[§3 execution boundary table](../../design/08-conventions-and-execution-boundary.md#which-convex-functions-may-touch-the-sandbox).** `setup` stays a **mutation** (resolve-only): `deriveCompactionDefaults` reads model caps from the **V8-safe** [`capabilities.ts`](../../../convex/providers/capabilities.ts) (`lookupCaps`, no AI SDK import) — `setup` does **not** become `"use node"` to derive the context window. The `compact` action is the only `"use node"` addition (it already is one, [`compact.ts:1`](../../../convex/engine/compact.ts)). The observer + its read query are **query-only** — never `"use node"`, never the box.
- **[04 — Compaction in the loop](../../design/04-durable-engine.md).** Threshold mode runs the next `llmStep` on the **compacted history** (no retry); `compact` appends a `CompactionEntry` as a **new leaf** ([`store.ts:171`](../../../convex/sessions/store.ts)), it does **not** mutate/delete prior entries — replay + audit need them. The frozen plan + cut-point keep the next decode's context byte-faithful (`responseMessages` round-trip preserved).
- **[06 — Observability (deferred)](../../design/06-phase-roadmap.md#observability-deferred).** The OTel observer is a **read-side consumer of the `events` table** + the per-step usage rollups — tracing is fed from durable state, **not** an in-loop dependency. No phase blocks on it; the engine never imports the observer.
- **[01 — success criteria](../../design/01-overview-and-goals.md#goals).** The sample agent demonstrates **G1** (interface fidelity: `createAgent`/`defineTool`/`defineAgentRegistry`), **G2** (realtime: `text_delta` via reactive query), **G3** (durable: survives a mid-turn replay), **G4** (sandbox parity, if it shells), **G5** (additive HITL: gated tool parks + resumes).
- **[§1 naming + §2 reference-header](../../design/08-conventions-and-execution-boundary.md#1-naming--namespace).** Every new identifier is `Cove*` (`createCoveOpenTelemetryObserver`); every span attribute is `cove.*`; `[cove]` prefix on every thrown error; every new file opens with the reference-header citing flue origin + `@opentelemetry/api`.

## Acceptance

1. **Auto-trigger fires exactly once (threshold).** A session whose context crosses `contextWindow − reserveTokens` invokes `compact` **exactly once**, appends a `kind:'compaction'` `sessionEntry` ([`store.ts:171`](../../../convex/sessions/store.ts)), and the **next** decode's context = `[summary + retained tail]` (`SessionHistory.buildContextEntries()`). Driven by `convex-test`/the `cove-test/mock` model; `decision.shouldCompact` flips true off the persisted per-step usage, not the chars/4 estimate.
2. **Mid-loop replay re-runs the journaled compact with no second summarization.** Killing the workflow between the `compact` step and the next `decode`, then resuming, returns the **cached** compaction journal result — `generateText` summarization is called **once** across the replay (§4.1). (The exhaustive crash-recovery E2E is [G2.6](phase-g2.6-convex-test-suite.md); G2.5 asserts the single-call invariant on the threshold path.)
3. **Disabled path.** A profile with `compaction:false` **never** sets `shouldCompact:true` (frozen `enabled:false`); `grep` confirms `setup` freezes `enabled:false` and `decode` short-circuits. An **explicit** `session.compact()` still works on a `compaction:false` session (overflow + explicit are independent of the threshold).
4. **`session.skill('review-pr')` resolves from the catalog (no box).** With the `review-pr` skill seeded via `importSkill`, `session.skill("review-pr")` returns a `PromptResponse` whose run carried the skill body as its prompt — **no box provisioned to resolve it** (the resolution is a `getSkill` query). An **unknown** skill → a typed `[cove]` error (`skill "<name>" not found in catalog`), not an FS miss.
5. **`session.compact()` appends an entry; no-op when nothing older.** An explicit `session.compact()` on a session with summarizable history appends a `CompactionEntry` and resolves `void`; on a session with nothing older than the tail it resolves `void` with **no** entry appended (the `{ compacted:false }` path, [`compact.ts:51`](../../../convex/engine/compact.ts)).
6. **Sample agent demonstrates G1–G5.** The E2E shows: **G1** `createAgent`/`defineTool`/`defineAgentRegistry` compile + register; **G2** `text_delta` streamed via the reactive `listForStream` query; **G3/G4** survives a simulated redeploy mid-turn (journal replay → coherent terminal, at-most-once model call per finalized step); **G5** the `postReview` gated tool parks an approval then resumes on `submitApproval`.
7. **OTel observer produces nested spans with correct error status + redaction.** A recorded `run_start..text_delta..tool..run_end` `CoveEvent[]` folded through `createCoveOpenTelemetryObserver` yields a **run root span** with nested **turn/tool/compaction** spans (parent linkage), the **failed-tool** span carries `SpanStatusCode.ERROR`, and with an `exportContent` redactor no content attribute (`cove.tool.result`/`cove.turn.input`) leaks. `grep -rn "flue\." convex/observability/` returns **nothing** (all `cove.*`).
8. **Green gates.** `tsc --noEmit` exits 0; `vitest run` green (OTel test + sample deterministic E2E + existing suite); `grep -rn "shouldCompact: false" convex/engine/` returns **0** (both [`decode.ts:189,199`](../../../convex/engine/decode.ts) hard-codes gone); `grep -rn "not_implemented" src/runtime/context.ts` no longer matches `session.compact`/`session.skill`.

## Risks & gotchas

- **The in-loop threshold must reuse persisted per-step usage, or it fires inconsistently vs the manual action.** The manual `compact` action computes `tokensBefore` from `estimateContextTokens` over the rebuilt messages ([`compact.ts:50`](../../../convex/engine/compact.ts) → `prepareCompaction`), which prefers the **last finalized assistant usage** ([`compaction.ts:108`](../../../src/runtime/compaction.ts) `estimateContextTokens` → `getLastAssistantUsageInfo`, [`compaction.ts:67`](../../../src/runtime/compaction.ts)). The in-loop `decode` threshold must read the **same** finalized `agentRequestSteps.usage` (not a fresh chars/4 estimate of partial text), or threshold-mode and the manual action disagree on when to compact. **Mitigation:** task 4 computes `shouldCompact` from `estimateContextTokens` over the **finalized** context (incl. the just-finalized assistant message with its persisted usage); the replay path uses the persisted row's usage so the branch is replay-stable.
- **`compact` mutates the active path mid-run — the next decode's context must stay byte-faithful.** A threshold compaction appends a `CompactionEntry` as a **new leaf** ([`store.ts:171`](../../../convex/sessions/store.ts)); the next `llmStep` rebuilds context from that leaf. If the cut-point or the `responseMessages` round-trip drifts, the model sees a malformed history. **Mitigation:** `compact` appends (never deletes) — `buildContextEntries()` walks `[summary + tail]`; the frozen `keepRecentTokens` cut-point is computed by the already-tested pure `prepareCompaction`; replay re-yields the **journaled** summary (a `step.runAction` cache), never a fresh `generateText`.
- **`setup` must not become `"use node"` to read the context window.** `resolveModel` is `"use node"` (imports `@ai-sdk/gateway`, [`gateway.ts:1`](../../../convex/providers/gateway.ts)) — calling it from `setup` (a mutation) would conflate the boundary. **Mitigation:** task 2 reads caps from the **V8-safe** `lookupCaps` ([`capabilities.ts`](../../../convex/providers/capabilities.ts), no AI SDK import); `setup` stays a mutation ([08 §3 table](../../design/08-conventions-and-execution-boundary.md#which-convex-functions-may-touch-the-sandbox)).
- **`session.skill()` must route through the catalog only — no sandbox FS walk** ([08 §3](../../design/08-conventions-and-execution-boundary.md#skills-resolve-at-the-call-site-not-in-the-sandbox)). flue resolved skills from the filesystem; re-introducing an `env.readFile` here would break replay-stability + the boundary. **Mitigation:** `submitSkill` calls `getSkill` (a query) and submits the body as a prompt; the sample's `review-pr` skill is seeded via `importSkill` (host-supplied `SKILL.md`), never read from a box.
- **Unknown model ⇒ `contextWindow:0` ⇒ threshold silently off (intended, but easy to mistake for a bug).** `lookupCaps` returns `undefined` for an uncatalogued id → `zeroMetadataCaps()` (`contextWindow:0`) → `shouldCompact` returns `false` for `contextWindow<=0` ([`compaction.ts:129`](../../../src/runtime/compaction.ts)). This is correct (skip threshold, keep overflow + explicit), but a test on an uncatalogued model will never auto-compact. **Mitigation:** the sample + the auto-trigger test use a catalogued model (or a `cove-test/mock` with an injected `contextWindow` + a low `reserveTokens` so the demo actually compacts).
- **Overflow→retry must not double-charge (and is only seeded here, not proven).** The overflow seam re-enters the **same** `stepNumber`; the `compact` step + the re-decode are journaled, so replay de-dups the model call. Treating the retry as a fresh `stepNumber` would double-charge. **Mitigation:** task 8 keeps the same `stepNumber` + a bounded overflow-retry cap; the full E2E proof (call-count 1 across a forced replay) is [G2.6](phase-g2.6-convex-test-suite.md)'s `tests/e2e/compaction.test.ts`.
- **The OTel observer is read-side — pin the span mapping to the `CoveEvent` contract or P9 field drift breaks it.** The observer reads `events.data` ([`schema.ts:329`](../../../convex/schema.ts)) shaped as `CoveEvent` ([`types.ts:923`](../../../src/runtime/types.ts)); if a P9/G2.1 emitter renames a correlation id (`turnId`/`operationId`) the `toolKey`/`compactionKey`/parent-linkage silently mis-nests spans. **Mitigation:** the observer imports the `CoveEvent` type (compile-time pin); the OTel test fixtures are built from the **same** event variants the engine emits; acceptance bar 7 asserts the nesting on a recorded sequence.
- **The live G1–G5 demo needs external resources the deterministic suite does not.** A real run needs a deployable Convex deployment + a provider key (or `cove-test/mock`) + an `@upstash/box` token if it shells, and (for the registered-agent path) [G2.4](phase-g2.4-cli-codegen.md)'s codegen to install the registry. **Mitigation:** the deterministic E2E (compaction auto-trigger, facade routing, OTel fold, HITL park/resume) runs on `convex-test`/the mock model; the README documents the live path; G2.4 is a *soft* dep (the agent compiles + registers without it).
- **`@opentelemetry/api` promotion is a no-op download but a real `package.json` change.** It is already in `node_modules` transitively ([`package-lock.json:658`](../../../package-lock.json), `1.9.0`); the only change is moving it to direct `dependencies` so the import is declared, not incidental. **Mitigation:** task 1 pins `^1.9.0`; no lockfile churn beyond the dependency-graph annotation.
- **`compaction:false` on the profile vs `enabled` on the frozen settings.** The author field is `compaction?: false | CompactionConfig` ([`types.ts:278,309,336`](../../../src/runtime/types.ts)); the engine's pure settings carry `enabled: boolean` ([`compaction.ts:24`](../../../src/runtime/compaction.ts)). Confusing the two (e.g. freezing `compaction: false` as the *config* rather than `enabled:false`) would silently re-enable the threshold. **Mitigation:** `setup` maps `compaction === false` → `{ enabled:false }`; `decode`/`shouldCompact` gate on `settings.enabled` ([`compaction.ts:128`](../../../src/runtime/compaction.ts)).
