# G2.6 — `convex-test` store-contract harnesses + full-stack E2E crash-recovery & throughput gate

> *Completes the [P12](../group-1/phase-12-compaction-tests.md) test strand (acceptance bars 3/4/6). Group-1 P12 landed the substance as **DI-unit tests** — 200 green vitest specs on the node env, the byte-stable `testModel.ts` seam ([`testModel.ts:103`](../../../convex/providers/testModel.ts)), and the result-path / replay-guard / throughput logic proven against injected ports ([`__tests__/loop.test.ts`](../../../convex/engine/__tests__/loop.test.ts), [`deltaBatcher.test.ts`](../../../convex/engine/__tests__/deltaBatcher.test.ts)). What was deferred is the **full-stack proof against the live Convex adapters**: `convex-test` is not installed, there is no `tests/` tree, and the three store-contract harnesses, the multi-turn crash-recovery E2E, the compaction E2E, and the production-cadence throughput gate were left as `convex run`-only. G2.6 installs `convex-test` + `@edge-runtime/vm`, wires a multi-project vitest config, ports flue's **SessionStore-half** store contract (omitting `AgentSubmissionStore` per M6), and adds three E2E/perf specs that drive the real Convex functions in an in-memory deployment — model injected at the `resolveModel` seam, no live provider and no live box. After this phase `npm test` is a single hermetic gate that proves durability end-to-end, not just per-port.*

## Goal & scope

The capstone verification phase. It moves Convex-function verification from `convex run` live to a hermetic, reproducible suite **and** lands the four bars P12 explicitly deferred to a consolidated gate ([06 §"Per-phase vs at-P12"](../../design/06-phase-roadmap.md), the P12 "Test suite" block, lines 449-475): the **three store-contract harnesses**, the **E2E multi-turn + crash-recovery** test, the **compaction E2E** (both modes), and the **throughput regression** ("re-run at P12 as a regression gate"). Two strands:

- **Installability + config.** Add `convex-test` + `@edge-runtime/vm` devDeps and a **multi-project** vitest config: the existing 200 pure-logic units keep the `node` env (they import `"use node"` modules transitively and have no DB), the new `tests/` tree runs on the `edge-runtime` env (`convexTest(schema)` runs functions inside an edge-runtime VM). One shared env fails — keep them split.
- **The hermetic suite.** Three store-contract harnesses (`session-store`, `run-store`, `event-stream-store`) porting flue's `defineStoreContractTests` **SessionStore half**, each with a `convex-test` adapter satisfying the contract against the live Convex mutations/queries; the multi-turn crash-recovery E2E (interrupt between `step.run*` seams, resume, assert coherent terminal + idempotent tool results + at-most-once model call per finalized step); the compaction E2E (threshold auto-compact + overflow→retry-same-step, `generateText` count 1 across a forced replay); and the throughput perf spec (≥200 deltas coalesce into far fewer in-position patches, committed before the workflow advances).

**Non-goals / scope cuts:**

- **No new engine wiring.** The overflow→retry mode — `isContextOverflow` in [`retry.ts`](../../../convex/engine/retry.ts) + the same-`stepNumber` overflow hook in `llmStep`/`runHandler` — is **owned by [G2.5](phase-g2.5-glue-sample-otel.md)** (its task 8, "Land the overflow classifier (seam only)"). G2.6 **only tests it**; it must not re-add the classifier or the hook (avoid duplicate wiring). Same for the threshold auto-trigger (G2.5 tasks 2-7) and the `compact`-as-loop-step plumbing.
- **No fourth store harness.** The flue `AgentSubmissionStore` contract (lease / attempt-marker / turn-journal / stream-chunk / claim assertions, [`define-store-contract-tests.ts:200-1005`](../../../../flue/packages/runtime/src/test-utils/define-store-contract-tests.ts)) is **not** reconstituted — its machinery is dropped (D5: `@convex-dev/workflow` owns recovery) and its surviving admission invariants were asserted in P6. Record the deliberate non-port with an M6 comment (acceptance greps for it).
- **No "use node" execution inside the VM where it cannot run.** The edge-runtime VM does **not** run real `"use node"` actions (`llmStep`/`dispatchTools`/`compact` are `"use node"`). The model is injected at the `resolveModel` seam (`cove-test/mock`, [`gateway.ts:38`](../../../convex/providers/gateway.ts)); tools use the framework/result/local-bash path so no box or network egress; any node-only path the VM can't execute is tested **pure** in `src/runtime`/`convex/engine` directly (the existing 200 units already cover the decode/dispatch cores) and `convex-test` is reserved for DB-contract + orchestration behavior.
- **No sample agent, no OTel, no auto-trigger wiring** — those are G2.5. No new providers, channels, or MCP.

## Hardened-contract obligations

- **[08 §4.1 — `llmStep` replay determinism *(critical)*](../../design/08-conventions-and-execution-boundary.md#41-llmstep-replay-determinism-critical).** The crash-recovery E2E is the full-stack proof of "the model is called **at most once per *finalized* step**": the test's model-call counter must read 1 per finalized `stepNumber` across an interrupt+resume, exercising the [`runDecode`](../../../convex/engine/decode.ts) replay guard (`loadStep().isFinalized → reconstructDecision`, [`decode.ts:86`](../../../convex/engine/decode.ts)). The compaction overflow path must show `generateText` count 1 across a forced replay — the `compact` step and the re-decode are journaled `step.run*` checkpoints, so a replay returns the cached compacted leaf + cached re-decode and never double-charges.
- **[08 §4.5 — tool rebuild + dispatch idempotency](../../design/08-conventions-and-execution-boundary.md#45-tool-rebuild-from-frozen-descriptors).** The crash-recovery E2E asserts the `appendToolResult` idempotency key (`toolCallId`) de-dups: a replayed `dispatchTools` returns the persisted result, never re-executes. This is observable as exactly one `toolResults` entry per `toolCallId` ([`steps.ts:152` appendToolResult filters by toolCallId](../../../convex/engine/steps.ts); [`dispatchTools.ts:121` skips already-resulted calls](../../../convex/engine/dispatchTools.ts)).
- **[08 §4.6 — streaming commit & subscription semantics](../../design/08-conventions-and-execution-boundary.md#46-streaming-commit--subscription-semantics).** The throughput spec asserts deltas are **coalesced into the step row** (not an append log) and the flush mutation **commits before the workflow advances**. Production cadence is `deltaBatchMs = 400`, `deltaBatchChars = 480` — the spec re-confirms the value chosen by the P4 stress and the in-position patch ([`steps.patchStreaming`](../../../convex/engine/steps.ts) patches `text`/`reasoning` on the same row).
- **[08 §4.7 — usage & cost](../../design/08-conventions-and-execution-boundary.md#47-usage--cost).** The compaction-threshold assertion reads the **widened** usage rollup (`cacheRead/cacheWrite/cacheWrite1h + cost{}`, [`schema.ts:76` usageValidator](../../../convex/schema.ts)) — the threshold math is computed off the full rollup, not a token-only subset, so the scripted usage must populate the cache fields when forcing a threshold crossing.
- **[04 — Compaction in the loop](../../design/04-durable-engine.md).** Both modes preserved and tested distinctly: proactive threshold (no retry, next decode on compacted history) and reactive overflow (retry the **same** `stepNumber`). `isRetryableModelError` (transient, [`retry.ts:31`](../../../convex/engine/retry.ts)) is a **separate** path from overflow-compact-retry — the E2E must not conflate them.
- **M6 / [D5](../../design/07-risks-and-decisions.md) — exactly three harnesses.** The suite has Session/Run/EventStream and **no** `AgentSubmissionStore` harness. `grep AgentSubmissionStore tests/` returns **only** the M6 non-port comment. Recovery is owned by `@convex-dev/workflow` (D5); flue's turn-journal/lease/attempt machinery is dropped, so its contract assertions die with it ([08 §5](../../design/08-conventions-and-execution-boundary.md#5-dropped--obsolete-explicit)).
- **[08 §2 — reference-header convention](../../design/08-conventions-and-execution-boundary.md#2-reference-header-convention).** Every new file under `tests/` opens with a header citing its flue/pi origin + npm package (or "New (Convex backend) · @cove/runtime" for the config/util) and the key transformation.

## Risks & gotchas

- **`convex-test` offline-installability is THE gating blocker.** Not in `node_modules`, not in `package.json`; the current sandbox has no registry access. **Mitigation:** install + commit `package.json` + `package-lock.json` on a networked machine so the offline env resolves from the lockfile, or vendor `convex-test` + `@edge-runtime/vm` into `node_modules`. If neither is possible, the harnesses + E2Es stay `convex run`-only and this phase cannot fully land — flag it before starting (this is roadmap open Q1).
- **The edge-runtime VM does not run real `"use node"` actions.** `llmStep`/`dispatchTools`/`compact` are `"use node"` ([`llmStep.ts:1`](../../../convex/engine/llmStep.ts), [`dispatchTools.ts:1`](../../../convex/engine/dispatchTools.ts), [`compact.ts:1`](../../../convex/engine/compact.ts)). **Mitigation:** inject the model at the `resolveModel` seam (`cove-test/mock`, [`gateway.ts:38`](../../../convex/providers/gateway.ts)) so no live provider; use framework/result/local-bash tools so no box/network egress; where the VM cannot execute a node leg, test that leg **pure** in `src/runtime`/`convex/engine` (the existing 200 units cover the decode/dispatch cores) and reserve `convex-test` for the DB-contract + orchestration behavior. The crash-recovery and compaction E2Es therefore assert the **replay invariants on persisted rows** + the model-call counter, not necessarily a live `"use node"` re-execution.
- **One shared vitest env fails.** `convex-test` needs `edge-runtime`; the 200 units import `"use node"` modules and DI ports that expect node globals. **Mitigation:** the multi-project config (task 2) or a per-file `// @vitest-environment edge-runtime` pragma on each `tests/` file — never a single global env.
- **The crash test must interrupt at the correct `step.run*` seam.** Killing an arbitrary JS line proves nothing; only an interrupt **between** journaled checkpoints (after `llmStep` finalize / before `dispatchTools`, [`runHandler.ts:27-31`](../../../convex/engine/runHandler.ts)) exercises journal replay. **Mitigation:** drive the workflow step-by-step through the `convex-test` runner and stop between specific `step.run*` calls, then resume; assert the row state proves the replay path (`isFinalized` row → `reconstructDecision`, no second model call).
- **Overflow→retry must reuse the same `stepNumber`.** A fresh `stepNumber` double-charges the model and defeats the replay guard. **Mitigation:** the E2E asserts the re-decode happens at the **same** `stepNumber` and `generateText` count is 1 across the forced replay; if it sees `stepNumber+1`, the G2.5 seam is wrong — fail loudly (don't paper over it in the test).
- **Porting the 1006-line flue store contract is large and fiddly.** The `AgentSubmissionStore` half is the bulk of it and is **dropped** (M6) — porting only the `sessions` block (≈60 lines) keeps it tractable. **Mitigation:** lift the `sessions` describe + `sessionData()` fixture verbatim, reshape only the backend factory to the Convex adapter; resist re-adding the dropped assertions even to "pass" them (skip-with-comment is the rule, not a stub).
- **The `runs`/`events` tables are not fully written yet.** `convex/runs.ts` reads the `runs` table but the row writer is G2.4/D18; `convex/events/*` is G2.1. **Mitigation:** the run-store adapter maps onto whatever G2.1/G2.4 expose and `it.skip`s the row-writer-dependent assertions with a G2.4 pointer if the writer is absent at build; the event-stream adapter runs against the G2.1 `append`/`listForStream` path. Both harnesses fail loudly (not silently) if the upstream surface is missing.
- **Filename cross-link consistency.** This file is `phase-g2.6-convex-test-suite.md`; the sibling plans ([G2.1](phase-g2.1-events-sdk-react.md), [G2.5](phase-g2.5-glue-sample-otel.md)) and the [group-2 README](README.md) already link to it under exactly that name, and all such links resolve. **Watch:** if this file is renamed, update the G2.1/G2.5 cross-links and the README in the same change so the docs keep resolving.
